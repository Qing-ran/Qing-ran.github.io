# 1. 什么是爬虫？

网络爬虫（又被称为网页蜘蛛，网络机器人）就是模拟浏览器发送网络请求，接收请求响应，一种按照一定的规则，自动地抓取互联网信息的程序。
网络爬虫是一种按照一定的规则、自动的抓取万维网信息的脚本或者程序
简单来说：爬虫是用事先写好的程序去抓取网络上的数据，这样的程序叫爬虫
也就是说，爬虫是通过编写程序，模拟浏览器上网，然后让其去互联网上抓取数据的过程

# 2. 爬虫的分类

1. 通用爬虫
抓取系统重要组成部分。抓取的是一整张页面数据。
2. 聚焦爬虫
建立在通用爬虫基础上，抓取的是页面中特定的局部内容。
3. 增量式爬虫
监测网站中数据更新的情况，只会抓取网站中最新更新出来的数据。

# 3. 反爬机制与反反爬策略

# 3.1 反爬机制

门户网站可以通过制定相应的策略或者技术手段，防止爬虫程序进行网站数据的爬取。

# 3.2 反反爬策略

爬虫程序可以通过制定相关的策略或者技术手段，破解门户网站中具备的反爬机制，从而可以获取门户网站中相关的数据。

比如： robots.txt 协议（君子协议）

# 4. 爬虫的知识体系和相关工具

# 5.HTTP 协议：

概念：服务器和客户端进行数据交互的一种形式。
常用请求头信息：
- User-Agent：请求载体的身份标识
- Connection：请求完毕后，是断开连接还是保持连接

常用响应头信息
- Content-Type：服务器相应回客户端的数据类型

# 6. HTTPS 协议

安全的超文本传输协议
区别：数据加密

加密方式：
- 对称密钥加密
- 非对称密钥加密
- 证书密钥加密

# 7. HTTP 请求

请求方式：get 请求 /post 请求
get 请求：会将请求的参数放到地址栏（URL）/ 网址后面，不安全
post 请求：不会将请求的参数放到网址后面，相对安全

# 8. requests 模块

python 中原生的一款基于网络请求的模块，功能非常强大，简单便捷，效率极高
作用：模拟浏览器发请求。

如何使用：（requests 模块的编码流程）
- 指定 url
- 发起请求
- 获取响应数据
- 持久化存储
